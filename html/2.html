ilk önce main.c isminde bir dosya oluşturalım. bu uygulama sadece ekrana merhaba yazsın 
#include 
int main(int argc, char const *argv[])
{
    printf("merhaba\n");
    return 0;
}
bu kodu derlemek için gcc komutuna kaynak dosyayı parametre olarak vermelisin.gcc main.c komutu parametrede verilen dosyayı derleyip binary dosyayı oluşturur. derleme sonunda a.out isimli bir dosya oluşur. eğer dosya ismi belirtilmezse öntanımlı olarak çalıştırılabilir dosya a.out ismiyle oluşur. ./a.out komutuyla bu uygulamayı çalıştırabilirsin. gcc komutuna parametre vererek bazı şeyleri değiştirebilirsin. gcc --help bütün parametre seçeneklerini listeler.

-o

eğer derleme işlemi sonrasında oluşacak olan dosya ismi -c parametresi ile verilebilir. bu parametre kullanılmazsa dosya a.out olarak oluşur. mesela üstteki örnekte bunu denersek gcc main.c -o merhaba seklinde gcc komutunu kullanırsak derleme sırasında oluşan dosyanın ismi merhaba olur. -o parametresinin kullanılmadığı durumda dosya ismi a.out olur

-c

bu parametre kaynak dosyadan obje dosyasını oluşturur. obje dosyası çalıştırılabilir bir dosya değildir. çalıştırılabilir dosyayı oluşturmak için obje dosyası gcc ye tekrar parametre olarak verilmelidir. main.c dosyamız için deneyecek olursak 
gcc -c main.c  // obje dosyası oluşturur
gcc main.o     //obje dosyasından binary dosyayı oluşturur
derleme işlemini bu sekilde yapmanın avantajı birden fazla dosyadan oluşan programlarda derleme yapılırken sadece değişikliğin olduğu kaynak kod dosyasını derlemek yeterli olmasıdır. örneğin 3 dosyadan oluşan bir program şöyle derlenir gcc dosya1.c dosya2.c dosya3.c dosya1.c de bir değişiklik yapıldıktan sonra tekrar derleme yapıldığı zaman değişikliğin yapılmadığı dosya2.c ve dosya3.c dosyaları da gereksiz yere derlenmiş olur. eğer derleme ve linkleme işlemlerini ayırırsak işlem şu şekilde yapılırken 
gcc -c dosya1.c
gcc -c dosya2.c
gcc -c dosya3.c
gcc dosya1.o dosya2.o dosya3.o
yine dosya1 üzerinde bir değişiklik yapıldıktan sonra diğer 2 dosyanın obje dosyaları elimizde olduğu için onlarla ilgili bir işlem yapmamıza gerek yoktur. bu durumda derleme işlemi şu şekilde yapılır: 
gcc -c dosya1.c
gcc dosya1.o dosya2.o dosya3.o

-Wall

derleme sırasında oluşan hataların daha ayrıntılı bir sekilde gösterilmesi için kullanılır. bu hata mesajlari uygulamadaki sorunları çözmede kullanılacağı için -Wall önemlidir.
